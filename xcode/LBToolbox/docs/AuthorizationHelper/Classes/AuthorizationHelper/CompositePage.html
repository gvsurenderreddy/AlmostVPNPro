AuthorizationHelper


Abstract

AuthorizationHelper simplify typical tasks.

Discussion

AuthorizationHelper simplify typical tasks which require use of Authorization:

  1. setup setuid tool
  2. run tool/script/command as a root
Besides using appropriate Authorization APIs to run things as root, AuthorizationHelper can perform digest based validation to prevent tempering (see -registerDigest:forFile: below ).

NOTE: Only tool/script with registered digest could be run as root.

Unless absolute path has been used, AuthorizationHelper will look in appliction bundle for tools and scripts. It expect to find:

associate digest (MD5) with given file name 

- ( void ) registerDigest:
        (NSString *) digest forFile:

installSetUIDTool:/runTool:withArgs:asRoot:/runScript:withArgs:asRoot: methods will compute digest of the tools/script and compare it with registered value. Only if values match, tools/script could be run as root.

installSetUIDTool:ofApplication:forAll:

install tool and make it setuid 

- ( NSString *) installSetUIDTool:
        (NSString *) fileName ofApplication:
        (NSString *) applicationName forAll:
        (BOOL ) installForAll;

Unless installForAll is true, this method will try to install tool into "~/Library/Application Support". If it is impossible to install tool into "~/Library/Application Support", then "/Library/Application Support" will be used.

It is OK to call this method more then once. For example, to check where tool has been installed.

Parameter Descriptions
fileName
tool file name
applicationName
sub-folder name into which tool will be installed
installForAll
if false, will attempt to install in ~/Library/Application Support
method result
full path to installed tool

runTool:withArgs:asRoot:

runs tool with given args either as root or as user 

- ( NSString *) runTool:
        (NSString *) fileName withArgs:
        (NSArray *) args asRoot:
        (BOOL ) runAsRoot;

If this tool was installed as SETUID and runAsRoot is YES, then installed version of the tool will run. In this case AuthorizationHelper will not employ any additional authorization APIs ( no authorization dialog will pop-up ).

This method will wait until tool closes its stdout.

Parameter Descriptions
fileName
tool file name
args
array of scring to be passed to the tool as parameters
runAsRoot
it YES, then tool will be run as root
